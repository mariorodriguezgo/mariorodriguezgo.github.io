Sistema Local de Distribución de ISOs con Filebrowser

Proyecto individual — Salesians Sarrià

Ubuntu 24.04 Server Docker Filebrowser RAID 5 Webmin Linux

Problemática identificada

En el entorno educativo de sistemas en Salesians Sarrià, se detectó una limitación importante en la infraestructura de red: Cuando alumnos intentaban descargar simultáneamente una imagen ISO, la red WAN del centro se saturaba.

Consecuencias detectadas:

  • Descargas extremadamente lentas.
  • Pérdida de tiempo lectivo.
  • Consumo innecesario de ancho de banda externo.
  • Dependencia total de la conexión a internet.

Era necesario implementar una solución que permitiera distribuir imágenes ISO de forma local, rápida y segura, sin depender de internet durante las clases.

Descripción del Proyecto

Diseñé e implementé un servidor local de distribución de archivos utilizando Ubuntu 24.04 Server, Docker y File Browser.

La solución consistió en:

  • Un servidor dedicado con disco de 500GB para sistema operativo y contenedores.
  • Tres discos de 1TB configurados en RAID 5 para almacenamiento redundante.
  • Gestión del almacenamiento mediante Webmin.
  • Despliegue de Filebrowser en contenedor Docker con volúmenes persistentes.
  • Personalización visual con el branding institucional de Salesianos.
  • Configuración de roles diferenciados para profesores y alumnos.

El servidor almacena localmente las imágenes ISO, permitiendo que los alumnos las descarguen a velocidad de red interna, eliminando la carga sobre la WAN.

Características Principales

Almacenamiento redundante

  • RAID 5 con 3 discos de 1TB.
  • Tolerancia a fallo de un disco.
  • Separación entre sistema y datos.

Contenerización

  • Filebrowser desplegado en Docker.
  • Uso de volúmenes para persistencia de configuración.
  • Arquitectura modular y fácilmente replicable.

Gestión de usuarios y permisos

  • Profesor: subir, editar, eliminar y descargar archivos.
  • Alumno: solo descarga.
  • Control de acceso seguro y estructurado.

Personalización

  • Adaptación del branding con logo institucional.
  • Organización clara de carpetas por sistemas y versiones.

Optimización de red

  • Descargas internas a velocidad LAN.
  • Eliminación de saturación de red WAN.
  • Independencia de internet para recursos críticos.

Resultados Obtenidos

  • Eliminación de cuellos de botella en la red externa.
  • Reducción drástica del tiempo de descarga.
  • Mejora significativa en la eficiencia de las clases prácticas.
  • Sistema estable, seguro y escalable.
  • Infraestructura preparada para ampliar el repositorio de recursos.

El sistema cumplió completamente el objetivo planteado y resolvió un problema real de infraestructura en el aula.

  • Despliegue de servicios mediante contenedores Docker.
  • Gestión de volúmenes y persistencia de datos.
  • Diseño de arquitectura orientada a rendimiento y disponibilidad.
